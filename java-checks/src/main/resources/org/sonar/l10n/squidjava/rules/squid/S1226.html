While it is technically correct to assign to parameters, it is better to introduce temporary variables to store intermediate results. This rule will typically detect cases where a constructor parameter is assigned to itself instead of a field of the same name, i.e. when this. was forgotten. Some developers might also expect assignments of method parameters to be visible from callers, which is not the case and can confuse them. All parameters should be treated as final.

Noncompliant Code Example

class Employee {
  public String name;

  public Employee(String name) {
    name = name;                  // Noncompliant - useless identity assignment
  }
}

Compliant Solution

class Employee {
  public String name;

  public Employee(String name) {
    this.name = name;             // Compliant
  }
}

Noncompliant Code Example

public int add(int a, int b) {
  a = a + b;                      // Noncompliant - the original argument is not modified
  return a;
}

public static void main(String[] args) {
  MyClass foo = new MyClass();
  int a = 40;
  int b = 2;
  foo.add(a, b);
  System.out.println("a = " + a); // Will print "40", not "42", which might seem confusing
}

Compliant Solution

public int add(int a, int b) {
  return a + b;                   // Compliant
}

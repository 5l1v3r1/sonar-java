Non-static initializers is a rarely used Java feature which can be confusing to most developers. When possible, they should be refactored into standard constructors or field initializers.

The following code:

public class MyClass {

  private final Date date;

  {                                      // Non-Compliant
    date = new Date();
  }

}

should be refactored into:

public class MyClass {

  private final Date date = new Date();  // Compliant

}

According to the Java Code Conventions as defined by Oracle, the parts of a class or interface declaration should appear in the following order in the source files:

  • Class (static) variables: first the public class variables, then the protected, then package level (no access modifier), and then the private.
  • Instance variables: first the public class variables, then the protected, then package level (no access modifier), and then the private.
  • Constructors
  • Methods

Noncompliant Code Example

public class Foo{  // Noncompliant
  private int field = 0;
  isTrue() {...}
  public Foo() {...}
  public static final int OPEN = 4;
}

Compliant Solution

public class Foo{  // Compliant
  public static final int OPEN = 4;
  private int field = 0;
  public Foo() {...}
  public boolean isTrue() {...}
}

On a Thread instance, methods wait(...), notify() and notifyAll() are available only because all classes in Java extends Object and so automatically inherit from those methods. There are two very good reasons to not call those methods on a Thread instance :

  • This is really confusing because what is really expected for instance when calling the wait(...) method on a Thread instance ? To suspend the execution of the Thread or to really wait for the acquisition of the monitor on this object ?
  • Internally, the JVM relies on those method calls to change the state of the Thread (BLOCKED, WAITING, ...), so calling those methods will corrupt the behavior of the JVM

Noncompliant Code Example

Thread myThread = new Thread(new RunnableJob());
...
myThread.wait(2000);

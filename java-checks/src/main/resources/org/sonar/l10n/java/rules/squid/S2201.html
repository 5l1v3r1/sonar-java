A non-void return value typically means one of two things:

  • the returned value indicates the success/status of the method
  • the returned value is the result of the method operations.

Thus, ignoring a return value is at best a waste of operations, and at worst a serious error. In either case, it is likely a bug.

Noncompliant Code Example

public void doSomething(String arg) {

if (isOkay(arg)) {
  try {
    arg.toLowerCase(); // Noncompliant; result of method thrown away

    File file = new File(arg);
    file.createNewFile();  // Noncompliant; result indicates operation success
    // use file

Compliant Solution

public void doSomething(String arg) {

if (isOkay(arg)) {
  try {
    String fileName = arg.toLowerCase();

    File file = new File(arg);

Exceptions

Fluent APIs are excluded from this check.

class A {
  A fluentMethod(String s) {
    //...
    return this;
  }

  public void doSomething() {
    A a = new A();
    a.fluentMethod(s);
  }
}
